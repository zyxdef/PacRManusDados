Pacotes no R

O R e os seus Pacotes

O software estatstico livre, gratuito e open source R possui em 04/07/2016 mais de 8700 pacotes ou bibliotecas (packages) de funes e rotinas disponveis em cdigo fonte e cdigo executvel no repositrio pblico de pacotes do site Comprehensive R Archive Network (CRAN).

https://cran.r-project.org/web/packages/

Cada pacote uma contribuio voluntria de pesquisadores e desenvolvedores do planeta inteiro e implementa algum conjunto especfico de funcionalidade.

IMPORTANTE: Antes de tentar implementar na unha algum mtodo, verifique se j no existe um pacote que execute o mtodo de que voc precisa.

Funcionalidades de um Pacote

As funcionalidade que cada pacote implementa podem ser

  • uma ou mais tcnicas analsticas estatsticas
  • um ou mais conjuntos de dados de acesso pblico
  • um ou mais mtodos grficos
  • um ou mtodo de acesso a dados externos ao R, podendo ser
    • formatos proprietrios de outros programas, estatsticos ou no
    • acesso a dados na internet
  • entre outros

Pacotes Padro e Adicionais

Cerca de 50 pacotes bsicos so instalados automaticamente durante a instalao do R no computador.

Os outros cerca de 8700 pacotes devem ser instalados individualmente conforme o interesse ou necessidade do usurio por mais funcionalidades.

Utilizao dos Pacotes

Dos cerca de 50 pacotes instalados automaticamente com o R, apenas 6 so carregados automaticamente quando o R iniciado.

Todos os outros pacotes instalados automtica ou manualmente no R precisam ser carregados manualmente pelo usurio para que suas funcionalidades sejam utilizadas durante a execuo do programa R.

Utilizando Pacotes

Uma vez carregado um pacote, todas as suas funcionalidades continuam disponveis para o usurio at o fim da execuo do R ou at voc descarregar o pacote manualmente da memria.

Assim, no necessrio carregar um pacote novamente na mesma execuo do R para utilizar as suas funcionalidades se o pacote j tiver sido carregado uma vez.

Vendo os Pacotes J Instalados

Execute o seguinte comando para ver o nome dos pacotes j esto instalados no R

rownames(installed.packages())

ou execute o seguinte comando para ver todas as informaes sobre os pacotes instalados,

installed.packages()

Se Voc Estiver Rodando o RStudio

Basta olhar a aba Packages para ver quais pacotes esto instalados.

Carregando e Descarregando Pacotes no R

Execute o seguinte comando para carregar um pacote no R

library(nome)

e execute o seguinte comando para descarregar um pacote no R

detach("package:nome", unload = TRUE)

em que nome sem aspas o nome do pacote j instalado.

Carregando e Descarregando Pacotes no R

Por exemplo, execute o seguinte comando para carregar as funcionalidades de modelagem de equaes estruturais que foram implementadas no pacote sem

library(sem)

e execute o seguinte comando para descarregar as funcionalidades de modelagem de equaes estruturais que foram implementadas no pacote sem

detach("package:sem", unload = TRUE)

Se Voc Estiver Rodando o RStudio

Basta selecionar ou desselecionar a caixa de seleo esquerda do nome do pacote na aba Packages.

Instalando e Desinstalando Pacotes

Execute o seguinte comando para instalar um pacote no R

install.packages("nome")

e execute o seguinte comando para desinstalar um pacote no R

remove.packages("nome")

em que "nome" entre aspas o nome do pacote.

Nota 1: Voc vai precisar ter conexo com a internet para executar a instalao.

Nota 2: Alguns pacotes exigem que os seus arquivos sejam compilados no Linux aps serem baixados, o que pode acarretar mais demora na instalao.

Instalando e Desinstalando Pacotes

NO EXECUTE ESTES COMANDOS NA SUA MQUINA! ELES SO S PARA ILUSTRAR!

Por exemplo, execute o seguinte comando para instalar e o pacote sem, que tem funcionalidades de modelagem de equaes estruturais

install.packages("sem")

e execute o seguinte comando para desinstalar o pacote sem, que tem funcionalidades de modelagem de equaes estruturais

Pipe

O Que Pipe?

Modo de organizar o trabalho com o conjunto de dados atravs do uso de pipe (tubulao ou encanamento), um conceito oriundo do Unix.

Muitas vezes, para obter os dados que queremos, precisamos realizar uma sequncia de operaes no conjunto de dados inicial:

  • A partir de um conjunto de dados (data frame) inicial, fazemos alguma operao ou transformao que gera um segundo conjunto de dados;
  • a partir desse segundo conjunto de dados, fazemos alguma outra operao ou transformao que gera um terceiro conjunto de dados;
  • e assim por diante, at obtermos o conjunto de dados que queramos no incio

Exemplo Didtico 1

Queremos calcular a mdia geomtrica dos valores 1, 2, 3 e 4

\[ G = \sqrt[n]{x_1 \cdot x_2 \dots x_n} \]

\[ G = \exp\left(\frac1n\sum_{i=1}^n\log(x_i)\right) \]

  • No o exemplo mais realista do mundo, mas permite explicitar como funciona o conceito de pipe.

Exemplo Didtico 1

Precisamos

  1. Calcular o logaritmo desses valores
  2. Calcular a mdia desses logaritmos
  3. Calcular a exponencial dessa mdia

possvel organizar de diversas formas o programa que vai realizar essas tarefas.

Exemplo Didtico 1

Implementao 1

    valores <- c(1, 2, 3, 4)
    logaritmos <- log(valores)
    mdia <- mean(logaritmos)
    mdia.geomtrica <- exp(mdia)
  • As etapas esto na ordem em que pensamos
  • No precisamos de nenhum dos valores intermedirios logaritmos e mdia

Exemplo Didtico 1

Implementao 2

    valores <- c(1, 2, 3, 4)
    mdia.geomtrica <- exp(mean(log(valores)))
  • Est mais sucinto
  • Mas as etapas esto na ordem inversa em que pensamos
  • Acabamos lendo de fora para dentro, o que torna mais difcil
    • escrever seu prprio cdigo,
    • ler o cdigo que outros escreveram,
    • ler o cdigo que voc mesmo escreveu h dois anos atrs

Exemplo Didtico 1

Implementao 3

    library(magrittr)   # para conseguir usar o operador %>%
    valores <- c(1, 2, 3, 4)
    mdia.geomtrica <- valores %>% log() %>% mean() %>% exp()
  • Est mais sucinto
  • As etapas esto na ordem em que pensamos
  • O que aconteceu aqui?
    1. O conjunto de dados dados serviu de entrada para log()
    2. O resultado de log() serviu de entrada para mean()
    3. O resultado de mean() serviu de entrada para exp()
    4. O resultado de exp() for armazenado em mdia.geomtrica

Exemplo Didtico 3

A partir de um conjunto de dados flight de voos partindo de um aeroporto, queremos as mdias dos atrasos de decolagem por hora para cada hora com mais de 10 decolagens com atraso.

Aqui, precisaremos

  1. separar somente os voos que decolaram com atraso
  2. agrupar esses voos por hora
  3. contar quantos voos atrasados ocorreram por hora
  4. calcular o atraso mdio para as horas que tiveram mais o que 10 decolagens atrasadas.

Para isso, utilizaremos as funes filter(), group_by() e summarise() do pacote dplyr, as quais veremos em diversas sees mais adiante.

Exemplo Didtico 3

Implementao 1

    temp2 <- group_by(temp1, date, hour)
    temp3 <- summarise(temp2, delay = mean(dep_delay), num = n())
    hourly_delay <- filter(temp3, num > 10)

Exemplo Didtico 3

Implementao 2

Como Funciona um Pipe?

O cdigo sem pipes

    sada.1 <- funo.1(dados, opes.1)
    sada.2 <- funo.2(sada.1, opes.2)
    resultado <- funo.3(sada.2, opes.3)

vira

    resultado <- 
        dados %>%
        funo.1(opes.1) %>% 
        funo.2(opes.2) %>%
        funo.3(opes.3)

com pipes.

Como Funciona o Operador %>%?

O resultado da expresso antes do %>% vira o primeiro argumento da funo depois do %>%.

valores %>% log()

o mesmo que

log(valores)

Como Funciona o Operador %>%?

O resultado da expresso antes do %>% vira o primeiro argumento da funo depois do %>%.

flights %>% filter(!is.na(dep_delay)) %>% group_by(date, hour)

o mesmo que

filter(flights, !is.na(dep_delay)) %>% group_by(date, hour)

que o mesmo que

group_by(filter(flights, !is.na(dep_delay)), date, hour)

Como Funciona o Operador %<>%?

Realiza todas as operaes e substitui o conjunto de dados original com o conjunto de dados resultante final

    dados %<>% 
        filter(!is.na(peso)) %>%
        filter(!is.na(estatura)) %>%
        mutate(imc = peso / estatura ^ 2) %>%
        select(-nome, -idade)

o mesmo que

    dados <-
        dados %>% 
        filter(!is.na(peso)) %>%
        filter(!is.na(estatura)) %>%
        mutate(imc = peso / estatura ^ 2) %>%
        select(-nome, -idade)

Quando Isso Fura

Em algumas situaes, o resultado de uma funo precisa ser passado como segundo argumento da funo seguinte. O que fazer?

  1. No usar pipes (soluo para ontem) ou
  2. Siviration
    • Leia a documentao dos pacotes magrittr e pipeR
    • Procure no Stack Overflow por exemplos de cdigo
    • S quando voce no estiver com a corda no pescoo
  3. Tentar o seguinte uso do operador %>%, com o ponto . indicando onde entrar a sada da funo anterior
    resultado <- dados %>%
        funo.1(opes.1, .) %>% 
        funo.2(opes.2a, ., opes2b)

read.table() o canivete suo para leitura de dados delimitados

Seus parmetros so:

read.table(
    file, header = FALSE, sep = "", quote = "\"'", dec = ".", 
    numerals = c("allow.loss", "warn.loss", "no.loss"),
    row.names, col.names, as.is = !stringsAsFactors, na.strings = "NA", 
    colClasses = NA, nrows = -1, skip = 0, check.names = TRUE, 
    fill = !blank.lines.skip, strip.white = FALSE, blank.lines.skip = TRUE,
    comment.char = "#", allowEscapes = FALSE, flush = FALSE,
    stringsAsFactors = default.stringsAsFactors(),
    fileEncoding = "", encoding = "unknown", skipNul = FALSE
)

Pacote utils

Alguns parmetros opcionais mais usados de read.table():

  • header indica se a primeira linha ser ou no o nome das colunas
  • sep = "" indica qual caractere ser o separador de colunas; se for uma string vazia (valor padro), ento espao, tabulao ou salto de linha sero delimitadores
  • dec indica qual caractere deve ser interpretado como ponto decimal
  • as.is indica se colunas no numricas devem ser armazenadas como dado textual (as.is = TRUE) ou se devem ser transformados em varivel categrica com a funo factor() (as.is = FALSE)
  • na.strings indica o vetor de strings que representam NA nos dados
  • nrows = -1 indica quantas linhas devero ser lidas no mximo
  • skip indica quantas linhas devem ser ignoradas antes de comear a ler

Pacote utils

Alguns parmetros opcionais mais usados de read.table():

  • fill indica se deve preencher com NA caso faltem colunas em alguma linha
  • strip.white indica se deve eliminar espaos em branco no incio e no fim de valores textuais
  • blank.lines.skip indica se deve ignorar linhas em branco ou se deve incluir uma linha inteira de NAs no data frame
  • encoding indica qual a codificao usada para gerar o arquivo; os valores mais usuais so:
    • "UTF-8" e "ISO-8859-1" ("LATIN1") na web e no Linux;
    • "UTF-16LE" no Windows; e
    • "MAC" no OS X

Pacote utils

  • tar(), zip(), untar() e unzip(): compactao e descompactao de arquivos
    • tar() e zip(): compactam arquivos
    • untar() e unzip(): extraem ou listam arquivos contidos em um arquivo compactado
    • tar() e untar(): lidar com formato .tar
    • zip() e unzip(): lidam com formato .zip

Pacote utils

# para compactar
tar(tarfile = "nome.tar", files = c("nome_1.ext", "nome_2.ext", ..., "nome_n.ext"))
zip(zipfile = "nome.zip", files = c("nome_1.ext", "nome_2.ext", ..., "nome_n.ext"))
# para listar o contedo de um arquivo compactado
untar(tarfile = "nome.tar", list = T)
unzip(zipfile = "nome.zip", list = T)
# para extrair arquivos de dentro de um arquivo compactado
untar(tarfile = "nome.tar", files = c("nome_1.ext", "nome_2.ext", ..., "nome_n.ext"))
unzip(zipfile = "nome.zip", files = c("nome_1.ext", "nome_2.ext", ..., "nome_n.ext"))
